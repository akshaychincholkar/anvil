# Delights — Help

*Screenshot folder: `docs/help/screenshots/delight/` — see numbering convention in [README.md](README.md).*

## What it is

Delights is a kanban board for the things you're looking forward to — books
to read, restaurants to try, places to visit — not obligations, just things
that sound fun. Each list moves items through **Want to → Enjoying →
Satisfied**, so anticipation has somewhere to live and completion gets its
own moment.

## How to use it

1. **Create a list.** Click **New list**, name it (e.g. "Restaurants to
   try"), and define up to 6 custom columns of your choosing — Text,
   Number, or Link type (e.g. "Cuisine," "Price," "Website").
   ![New list with custom columns](../../frontend/public/help/delight/01-new-list.png)
2. **Add items.** Use **Add** or the **+** on any lane to add a card — fill
   in your custom columns (Anvil suggests values you've typed before, so
   repeat entries like "Italian" autocomplete). Anvil suggests a status but
   you can put it straight in whichever lane fits.
3. **Drag between lanes.** Move a card from **Want to** → **Enjoying** →
   **Satisfied** as you go, or reorder within a lane by priority.
   ![Kanban board](../../frontend/public/help/delight/02-kanban-board.png)
4. **Close the loop when you finish.** Dragging (or adding) a card into
   **Satisfied** pops up a small completion prompt: the date (defaults to
   today) and an optional "How was it?" note — this is what turns "crossed
   off" into an actual memory you can look back on.
5. **Use a "tags" column to filter.** Name any column "tag" or "tags" and
   its comma-separated values automatically become filterable chips at the
   top of the board.

## Why it matters

Most productivity tools are built entirely around obligation — tasks,
deadlines, things you *have* to do. Delights is deliberately the opposite: a
dedicated place for things you *want* to do, tracked with the same care as
everything else.

- **Anticipation is a real form of well-being, and it needs a home.**
  Research on happiness consistently finds that looking forward to something
  contributes as much to wellbeing as the experience itself. If your only
  system is a to-do list, the fun stuff never gets written down anywhere —
  and what doesn't get written down quietly stops happening.
- **The three-lane structure ("Want to → Enjoying → Satisfied") mirrors how
  desire actually plays out**, instead of collapsing everything into a flat
  checkbox. Seeing something sit in "Enjoying" — actively happening — is
  its own small pleasure that a plain todo-done binary erases.
- **The completion note closes the loop instead of just crossing something
  off.** "How was it?" takes two seconds to answer but turns a finished item
  into a small archived memory instead of a checkbox that silently
  disappears. Months later, that note is the difference between remembering
  you *did* something and remembering how it actually *felt*.

Used consistently, Delights is meant to counterbalance a life run entirely by
obligation — a reminder, in the same app where you track discipline and
habits, that pursuing things purely because they sound good is also worth
doing on purpose.
